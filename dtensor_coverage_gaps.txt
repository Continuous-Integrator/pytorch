# DTensor Op Coverage Gaps
Cross-checked against actual registered sharding strategies, decomposition
tables, and TestMultiThreadedDTensorOps (393 passed, 44 skipped, 235 xfailed).
The CSV had some stale entries (14 ops marked as no-sharding that actually have
strategies, ~17 ops marked as blocked whose sub-ops now have strategies); both
are corrected here.
## Ops with decompositions: blocked by constructors (31)
These decompose into ops with no tensor inputs (arange, scalar_tensor, empty,
etc.), so PlacementTrackingMode can't propagate placements.
**13 of these can bypass the decomp entirely with direct strategies:**
| Op | Direct strategy |
|----|----------------|
| hardshrink, threshold, lcm, special_entr, special_xlog1py, exponential_ | pointwise (1 line each) |
| nansum | `LINEAR_REDUCTION_OP_MAP` as `"sum"` |
| cumprod | copy `cumsum_strategy` |
| select_scatter | scatter pattern in `_tensor_ops.py` |
| index_fill.int_Scalar | index pattern in `_tensor_ops.py` |
| renorm | norm along dim, reduction-like |
| linalg_cross | `single_dim_strategy` |
| grid_sampler_2d | batch-dim-only like pooling |
**18 need the decomp infrastructure fix:**
*12 pure constructors* (no tensor inputs, only relevant as sub-ops):
arange.start_step, empty_strided, eye, full, linspace, linspace.Tensor_Tensor,
logspace, ones, randn, normal.float_float, tril_indices, triu_indices, zeros.
*6 complex ops where the decomp is the natural strategy:*
_adaptive_avg_pool2d, _batch_norm_with_update, multilabel_margin_loss_forward,
im2col, normal.Tensor_Tensor, cumprod (if scan-dim sharding desired).
### Unblocking constructor ops in DecompShardingStrategy
The simplest fix: in PlacementTrackingMode.__torch_dispatch__, when no
DTensorSpec is found in args (i.e. a constructor), assign Replicate to the
output instead of raising NotImplementedError.
This is safe for all 14 scalar_tensor-blocked ops today — every call uses
either a Python literal constant (0, 1, -inf, etc.) or a shape read from a
non-sharded dimension. No op passes a sharded-size-derived value.
In general, assuming Replicate is not always correct: scalar_tensor(x.size(
sharded_dim)) should be Partial(sum), not Replicate. A principled future fix
is to use SymInts with sharding provenance during decomp tracing, so that
x.size(sharded_dim) returns a tagged SymInt that constructors can use to derive
the correct placement. This is analogous to how dynamic shapes track symbolic
relationships in torch.compile.
## Ops with decompositions: blocked by tensor ops (15)
Blocked by tensor ops that could get strategies: as_strided (4 ops),
_unsafe_index/index_put (5 ops), and constructor chains (6 ops). These have
complex index/stride logic — the decomp path is the right approach, the blocker
is adding strategies for `as_strided` and `_unsafe_index`.
| Op | Blocked by |
|----|------------|
| as_strided_copy, diagonal_scatter, native_group_norm, unfold, unfold_copy | as_strided |
| _upsample_nearest_exact1d, reflection_pad1d, replication_pad1d/2d, upsample_nearest1d | _unsafe_index, arange |
| max_unpool2d, max_unpool3d | _unsafe_index_put, arange |
| logspace.Tensor_Tensor | linspace.Tensor_Tensor (constructor chain) |
| multi_margin_loss | ne.Tensor*, arange, scalar_tensor |
## Ops with no decompositions and no sharding rules (73)
### Pointwise ops — 1 line each in _pointwise_ops.py (22 ops)
*Unary:* special_airy_ai, special_bessel_y0/y1, special_modified_bessel_i0/i1/k0/k1,
special_scaled_modified_bessel_k0/k1.
*Binary:* special_chebyshev_polynomial_t/u/v/w, special_hermite_polynomial_h/he,
special_laguerre_polynomial_l, special_legendre_polynomial_p,
special_shifted_chebyshev_polynomial_t/u/v/w.
*Also pointwise:* rrelu_with_noise, hash_tensor.
### Constructors — need special handling (4 ops)
scalar_tensor (high-impact, blocks 14 decomp ops), empty.memory_format,
empty_permuted, randint.
### Reductions/scans — existing helpers (8 ops)
| Op | Helper |
|----|--------|
| median, nanmedian (global) | `common_reduction_strategy`, replicate-only |
| nanmedian.dim | `common_reduction_strategy`, dim reduction |
| kthvalue, mode | `sort_strategy` (replicate on reduction dim) |
| cummax, cummin, logcumsumexp | `single_dim_strategy` (replicate on scan dim) |
### Pooling — batch-dim-only sharding (9 ops)
All use `single_dim_strategy` with Shard(0) only, matching existing
_adaptive_avg_pool2d. One shared helper covers all 9: _adaptive_avg_pool3d,
adaptive_max_pool2d/3d, avg_pool2d/3d, fractional_max_pool2d/3d,
max_pool2d/3d_with_indices.
### Index/scatter (6 ops)
*Replicate-only:* searchsorted, bincount, masked_scatter, put.
*Extend scatter_add pattern:* scatter_reduce.two, index_reduce — similar to
existing scatter_add in _tensor_ops.py but with reduction mode.
### Replicate-only — hard or niche ops (24 ops)
*Data-dependent shapes:* _unique2, unique_consecutive, nonzero, nonzero_static.
*Need all data:* _cdist_forward, _pdist_forward, _embedding_bag_forward_only,
grid_sampler_3d, max_pool2d_with_indices_backward, segment_reduce,
_histogramdd_from_bin_cts, histogram.bin_ct.
*Linalg:* _linalg_det, _linalg_slogdet, linalg_matrix_exp, allclose,
masked_select.
*Not meaningful:* _upsample_bilinear2d_aa, multinomial, resize_, resize_as_,
as_strided.
### Special cases
*addbmm* — could use `gen_single_dim_einsum_strategies` ("bmk,bkn->mn" + bias),
or more simply decompose as bmm + add (both already have strategies, so
DecompShardingStrategy would derive the combined strategy automatically).
## Recommended implementation order
| Step | What | Ops | Approach |
|------|------|-----|----------|
| 1 | Direct strategies for decomp-blocked ops | 13 | Bypass broken decomps with pointwise/reduction/scatter registrations |
| 2 | Pointwise ops (no decomp) | 22 | Add to `pointwise_ops` list |
| 3 | Replicate-only ops (no decomp) | 28 | `replicate_op_strategy`, one-liner each |
| 4 | Reductions + pooling (no decomp) | 17 | Existing helpers: common_reduction_strategy, sort_strategy, pooling helper |
| 5 | Constructor fix in PlacementTrackingMode | 18 | Assign Replicate to constructor outputs during decomp tracing |
| 6 | scatter_reduce, index_reduce strategies | 2 | Extend scatter_add pattern |
| 7 | as_strided / _unsafe_index strategies | 15 | Unblocks temporarily-blocked decomp ops |
Steps 1-3 cover 63 ops with minimal effort (mostly one-liners). Step 4 adds 17
more with existing helpers. Step 5 is a small infrastructure change that
cascades to 18 ops. Steps 6-7 are the hard tail.
